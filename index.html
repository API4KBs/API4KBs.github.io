API for Knowledge Platforms (API4KP)

Technical Specification and Implementation

The API4KP standard is based on a model-driven architecture that combines UML models, OWL ontologies and OpenAPI specifcations to define a platform, technology and vendor independent way to build Knowledge Based Systems.

Implementations of the Services are also provided (in build order):

  1. Language Service
    Functions for the syntactic manipulation of Knowledge Artifacts (e.g. parsing/serialization, validation, translation)  
  2. Knowledge Artifact Repository - JCR
    JCR-based persistence layer to support the storage and retrieval of knowledge artifacts 
  3. Knowledge Asset Repository
    Logical Repository with support for metadata, discovery and content negotiation 
  4. Inference Service
    Coming soon...

The following modules support the build process

  1. Configurable Maven Archetype
    Instantiates Mvn Archetypes using parameters from .properties files.
    Used to generate client and server modules for 'API Framework'
  2. Maven Enforcer Rules (optional)
    Validates quality criteria for 'master' branches

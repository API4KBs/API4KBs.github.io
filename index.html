API for Knowledge Platforms (API4KP)

Technical Specification and Reference Implementation

The API4KP standard is based on a model-driven architecture that combines UML models, OWL ontologies and OpenAPI specifcations to define a platform, technology and vendor independent way to build and interact Knowledge Based Systems.

Reference Implementations

Mayo Clinic

The Mayo Clinic "Knowledge Management and Delivery" Team has released a reference implementation of a "Knowledge Management and Delivery Platform" proof-of-concept.

The implementation consists of the following libraries:

  1. KMDP BOM 
    The "Bill of Materials" setting common dependencies
  2. API4KP Specification
    Packages and deploys the API4KP machine consumable documents
  3. KMDP Models
    Compiles the platform-independent specification into a Java-oriented platform-specific one
  4. KMDP Standard Models
    Java implementation of standard schemas, languages and metamodels
  5. KMDP Spring Maven Archetype
    Infrastructure for a Spring-based, ReSTful implementation of the APIs
  6. KMDP API Framework
    Further compiles the platform-independent API specification into a Java+Spring Web Service Framework

Reference implementations of the Services are also provided (in build order):

  1. Language Service
    Functions for the syntactic manipulation of Knowledge Artifacts (e.g. parsing/serialization, validation, translation)  
  2. Knowledge Artifact Repository - JCR
    JCR-based persistence layer to support the storage and retrieval of knowledge artifacts 
  3. Knowledge Asset Repository
    Logical Repository with support for metadata, discovery and content negotiation 
  4. Inference Service
    Coming soon...

The following modules support the build process

  1. Configurable Maven Archetype
    Instantiates Mvn Archetypes using parameters from .properties files.
    Used to generate client and server modules for 'API Framework'
  2. Maven Enforcer Rules (optional)
    Validates quality criteria for 'master' branches

Build Instructions

Requirements: JDK 8, Maven 3.3.9+

  1. Clone / install the configurable archetype plugin
    1. git clone https://github.com/API4KBs/maven-cfg-archetype-plugin.git
    2. cd maven-cfg-archetype-plugin
    3. (checkout develop) 
    4. mvn clean install
  2. Clone / install the API4KP specifications
    1. git clone https://github.com/API4KBs/api4kbs.git
    2. cd api4kbs
    3. (checkout develop) 
    4. mvn clean install
  3. Clone / install the master BOM 
    1. git clone https://github.com/API4KBs/kmdp-bom.git
    2. cd kmdp-bom
    3. (checkout develop) 
    4. mvn clean install
  4. Clone / install the core libraries
    1. git clone https://github.com/API4KBs/kmdp-models.git
    2. cd kmdp-models
    3. (checkout develop) 
    4. mvn clean install
  5. Clone / install the standard models
    1. git clone https://github.com/API4KBs/kmdp-standard-models-impl.git
    2. cd kmdp-standard-models-impl
    3. (checkout develop) 
    4. mvn clean install
  6. Clone / install the OpenAPI/Spring Archetype
    1. git clone https://github.com/API4KBs/kmdp-archetype.git
    2. cd kmdp-archetype
    3. (checkout develop) 
    4. mvn clean install
  7. Clone / install the API framework
    1. git clone https://github.com/API4KBs/kmdp-api.git
    2. cd kmdp-api
    3. (checkout develop) 
    4. mvn clean package -Prefresh
    5. mvn clean install
  8. Clone / install the Knowledge Artifact Repository implementation
    1. git clone https://github.com/API4KBs/kmdp-artifact-repository.git
    2. cd kmdp-artifact-repository
    3. (checkout develop) 
    4. mvn clean install
  9. Clone / install the Language / Transrepresentation implementation
    1. git clone https://github.com/API4KBs/kmdp-language.git
    2. cd kmdp-language
    3. (checkout develop) 
    4. mvn clean install
  10. Clone / install the Knowledge Asset Repository implementation
    1. git clone https://github.com/API4KBs/kmdp-asset-repository.git
    2. cd kmdp-asset-repository
    3. (checkout develop) 
    4. mvn clean install
